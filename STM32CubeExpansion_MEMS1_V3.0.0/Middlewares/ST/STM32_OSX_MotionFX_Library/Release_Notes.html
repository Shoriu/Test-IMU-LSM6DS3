Release Notes for OSX MotionFX Sensor Fusion Library

Copyright 2016 STMicroelectronics

 

What is MotionFX?

  • The OSX MotionFX (iNEMOEngine PRO) fusion suite is a filtering and predictive software. It uses advanced algorithms to integrate outputs from multiple MEMS sensors in a “smart” way, independent of environmental conditions, to achieve optimal performance. Real-time motion sensor data fusion is set to significantly improve the user experience, increasing accuracy, resolution, stability and response time in advanced motion-based applications in the consumer, computer, industrial and medical
    fields.

MotionFX features

  • 9-axis fusion (accelerometer + gyroscope + magnetometer)
  • 6-axis fusion (accelerometer + gyroscope)
  • Gyroscope bias calibration
  • Magnetometer calibration

Usage and build notes

  • The MotionFX engine is provided as a node-locked library which allows derivative firmware images to run on a specific STM32 Nucleo device only. Licensing activation codes has to be requested to ST and included in the project prior to attempting its usage - failure to accomplish this step will prevent proper API execution. The resulting firmware binary image will therefore be node-locked. For complete information about the ST OpenSoftwareX M1 limited license agreement please refer to the below "License" paragraph.

How to activate the node-locked license

  • The license request form has to be generated through the OSX_LicenseWizard tool (included with each Open.MEMS distribution package). Few steps are required to generate the request:
    • Connect an STM32 Nucleo board to the PC (via USB cable)
    • Run the OSX_LicenseWizard and generate the node and site codes
    • Edit the user information (name, company and email address)
    • Accept to generate and send the license request to STMicroelectronics
    • Once the license activation codes are received edit the contents of the osx_license.h file
      (note: also delete or comment the "#error...." line)
  • Include the osx_license.h file from the user code
  • Build the target project, download and run the firmware image
